# Memory dumps from a running Noita

These files capture state from a single Noita process so the comparison
scripts in `scripts/` can run without `noitrainer` (the sibling Go tool
that reads Noita memory) and without a live game.

| File | Source command | Schema |
| --- | --- | --- |
| `biome_flags.ndjson` | `noitrainer-cli biome-flags` | one line per loaded biome chunk: `{cx, cy, name, ptr, biomeDataPtr, wobbleEligible, wavyEdge, forceOriginal}` |
| `biome_at_resolutions.ndjson.gz` | `noitrainer-cli biome-at-many < coords.txt` | one line per probed world coord: `{wx, wy, gridWidth, gridHeight, xShift, yShift, origCX, origCY, subX, subY, original, resolved, wobbled, wobbleType, neighborDir, neighborCX, neighborCY}` — the coords were generated by `scripts/sample_coords.mjs --mode=edges` (84,060 points: corners + edges + a 16-px sub-grid in the wobble-active strips of every loaded chunk). |
| `pixel_scenes.ndjson.gz` | `noitrainer-cli pixel-scenes` | one line per pixel scene currently queued in the BiomeGrid: `{x, y, materialsFile, colorsFile, backgroundFile, flagSkipBiomeChecks, flagSkipEdgeTextures, index, vec}` (`vec` is `"main"` or `"alt"`). 323 entries from this snapshot session — the authoritative answer to "what scenes did the game place?" for diffing against telescope's `loadPixelScene`. |

`biome_flags.ndjson`, `biome_at_resolutions.ndjson.gz`, and
`pixel_scenes.ndjson.gz` all come from the same Noita session: build
dated 2025-01-25, seed `864604507`, NG+ count `0`, normal mode.
`biome_flags.ndjson` lists every biome chunk Noita allocated (1,868 are
named, the rest are `_EMPTY_` / `???` and excluded).
`biome_at_resolutions.ndjson.gz` is the wobble fixture used by
`scripts/compare_wobble.mjs`.

`pixel_scenes_session2.ndjson.gz` is a second pixel-scene dump from seed
`1590673099` (NG+ 0, same build), captured while investigating a paint
vs queue mismatch in the snowcave/HM transition. Used as an
additional fixture for `scripts/compare_pixel_scenes.mjs`.

To regenerate (requires noitrainer + a running Noita with the same
build/seed/NG+):

```sh
noitrainer-cli biome-flags > data/dumps/biome_flags.ndjson
node scripts/sample_coords.mjs --mode=edges < data/dumps/biome_flags.ndjson > /tmp/coords.txt
noitrainer-cli biome-at-many < /tmp/coords.txt | gzip > data/dumps/biome_at_resolutions.ndjson.gz
noitrainer-cli pixel-scenes | gzip > data/dumps/pixel_scenes.ndjson.gz
```

The comparison scripts read these dumps directly (`.gz` is decompressed
transparently). To use a different dump, pass `--input=PATH`.
